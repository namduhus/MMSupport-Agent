# PRD
한국에 장기 거주하는 외국인들은 응급처치가 필요한 상황에서 복합적인 어려움을 겪는 문제.

최종 수정일: 26.01.25(일)

## Multilingual Medical Support Agent for Foreign Residents in Korea

## Problem

1. **응급 상황에서의 정보 공백**
   - 즉시 해야 할 응급처치 순서를 몰라 대응이 지연됨
   - 상황별 주의사항(금기사항) 미인지로 위험 증가
   - 신뢰 가능한 공식 정보 접근이 어려움

2. **언어 장벽으로 인한 응급처치 오해**
   - 응급처치 용어와 증상 설명의 이해 부족
   - 번역 품질 문제로 핵심 지침이 왜곡될 가능성
   - 잘못된 해석으로 부적절한 처치 수행 위험

3. **문화적/지역적 차이로 인한 혼란**
   - 응급 대응 문화와 절차에 대한 경험 차이
   - 현지 응급 연락 체계(119 등)와 안내 방식 이해 부족
   - 주변 도움 요청 방식에 대한 혼선

이러한 문제는 의료 서비스 접근성을 낮추고, 외국인 대상 서비스의 신뢰도를 저하시킴.

---

## Solution Overview

RAG 기반으로 응급환자 매뉴얼 정보를 활용하여 정보를 제공하는 Agent 시스템을 구축.

- **핵심 기능**: 다국어(한국어, 중국어, 베트남어, 태국어) 사용자 입력을 기반으로 응급환자 매뉴얼 정보를 검색하고 이해하기 쉬운 형태로 제공
- **문화적 맥락 반영**: GraphDB를 활용해 문화적 맥락을 고려한 의미 조정 레이어 포함

## Goals

### Business Goals

- 외국인 대상 의료 AI 서비스 신뢰도 확보
- 향후 보험 연계 또는 유료 모델 전환 기반 마련

### User Goals

- 다국어(한국어, 중국어, 베트남어, 태국어)로 구성이 되어 해당 나라별 언어로 응급처치 정보 획득 가능
--- 

### System Flow

1. 사용자 언어 입력
2. 응급처치 관련 문서를 벡터로 변환
3. Qdrant에서 유사 문서 검색
4. GraphDB로 문화적 맥락을 반영해 의미 조정
5. 검색 결과를 통합해 응급환자 매뉴얼 가이드를 생성
6. 사용자 언어로 최종 응답 제공
---

### LLM & Framework

- **LLM Model**: `ministral-3:8b` (Ollama)
- **Framework**: LangChain V1.0 + Ollama (`langchain-ollama`)
- **DataBase**: Qdrant(Vector DB) + Neo4J(GraphDB)
